## 错误分类和修复策略

### 1. 可选属性访问 (19个错误)

**错误类型**: `TS18048: 'X' is possibly 'undefined'`

**修复方法**: 使用可选链和空值检查，**不用默认值**

```typescript
// ❌ 错误: 使用默认值
const opacity = config.opacity ?? 1.0;

// ✅ 正确: 使用空值检查
if (config.opacity !== undefined) {
  material.opacity = config.opacity;
}

// 或使用可选链
const opacity = config.opacity;
if (opacity != null) {
  material.opacity = opacity;
}
```

**影响文件**: app.ts (约15处)

### 2. 函数参数不匹配 (10个错误)

**错误类型**: `TS2554: Expected X arguments, but got Y`

**修复方法**: 更新函数签名或调用方式

```typescript
// 示例: requireElement 签名
// 错误: requireElement(root, selector, true)
// 修复: requireElement(root, selector) // 移除第3个参数
```

**影响文件**: app.ts, viewer-utils.ts

### 3. 类型赋值问题 (101个错误)

**错误类型**: `TS2345, TS2322, TS2362, TS2365` 等

**修复方法**: 
- 使用类型断言 (`as Type`)
- 添加空值检查
- 修复类型定义使其匹配实际使用

**主要场景**:
- `unknown` 类型不能直接赋值 (约5个)
- `number | undefined` 不能赋给 `number` (约4个)
- 算术运算操作数类型错误 (约5个)
- 索引访问类型问题 (约10个)
- 其他类型不匹配 (约77个)
